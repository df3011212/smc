除個別檔案另有明確授權外，網站內容可免費供個人學習與非商業研究使用，但著作權與署名權仍受保留。

1. 內容來源原則

網站管理者應僅上傳自行創作、已取得合法授權、屬於公有領域，或依法得合理使用的內容。不得因為網路上可以找到、可以下載，或標示免費，就推定可以任意重製、散布或重新上架。

2. 免費使用範圍

除檔案內另有授權條款外，使用者可免費：

  • 下載並保存一份或合理數量的副本供個人學習。
  • 在非商業、非冒名且保留來源的情況下引用少量內容。
  • 依教材進行回測、模擬、研究與個人程式修改。

未經另外同意，不得轉售、收費再包裝、建立付費會員庫、冒名發布、移除作者與版本資訊，或宣稱為官方授權課程。

3. 程式碼與個別檔案授權

若某一份程式碼、手冊或壓縮檔內附有獨立授權條款、開源授權或第三方授權，應優先依該檔案的條款使用。不同檔案可能有不同的修改、散布、署名或商業使用條件。

4. 商標與平台名稱

TradingView、Pine Script、GitHub，以及其他公司、產品、交易所與服務名稱均可能是其權利人的商標。本網站僅在描述相容性、教學對象或技術環境時使用相關名稱，不表示取得官方認證或合作。

5. 權利通知應提供的資料

若你是權利人或獲合法授權的代理人，認為網站內容侵害著作權、商標、個人資料或其他權利，請透過本網站所屬 GitHub Repository 的 Issues 或站長另行提供的聯絡方式提交通知，並包含：

  1. 通知人姓名或單位名稱，以及可回覆的聯絡方式。
  2. 所主張權利的作品、商標、資料或其他權利之具體說明。
  3. 涉嫌侵權內容的完整網址、檔名、頁面區塊或可定位資訊。
  4. 足以說明權利歸屬或代理權限的資料。
  5. 你希望採取的處理方式，例如更正署名、移除特定檔案或暫時下架。
  6. 聲明所提供資料真實，且通知係基於善意提出。

請勿在公開 Issue 中張貼身分證、銀行帳戶、私鑰、密碼或其他不必要的敏感資料。

6. 處理流程

收到可辨識的通知後，網站管理者可先暫時停用爭議內容或下載連結，並檢視來源、授權與雙方說明。若通知資料不足，可能要求補充。若爭議無法單純確認，相關內容可維持暫時下架，直到權利狀況較為明確。

7. 惡意或不實通知

請勿利用權利通知制度騷擾、冒名、打擊競爭者或要求移除不屬於你的內容。通知人應自行確認其權利基礎與陳述真實性。

8. GitHub 平台程序

本網站部署於 GitHub Pages。權利人除可先聯絡網站管理者外，也可以依 GitHub 公布的內容移除、著作權或私人資訊申訴程序處理。GitHub 是否移除內容，仍由平台依其政策與收到的資料判斷。
